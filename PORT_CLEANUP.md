# 端口清理功能说明

## 功能概述

系统内置了自动端口清理功能，在启动服务前会自动检查并清理端口4080的占用进程。

## 工作原理

### Windows系统
- 使用 `netstat -ano` 命令查找占用端口的进程
- 使用 `taskkill /F /PID <pid>` 强制终止进程

### Linux/Mac系统
- 使用 `lsof -ti:<port>` 命令查找占用端口的进程
- 使用 `kill -9 <pid>` 强制终止进程

## 使用方法

### 自动清理（推荐）
```bash
npm run dev
```
此命令会在启动服务前自动清理端口占用。

### 手动清理
```bash
npm run kill-port
```
此命令只清理端口占用，不启动服务。

## 端口说明

- **4080**: 统一端口（前端和后端）

## 注意事项

1. 端口清理会强制终止占用端口的进程，请确保没有其他重要服务在使用这些端口
2. 如果端口清理失败，请手动检查并终止相关进程
3. 生产环境部署时，建议使用不同的端口或配置反向代理

## 故障排除

### 端口仍然被占用
如果端口清理后仍然无法启动服务，请尝试：

1. 手动检查端口占用：
   ```bash
   # Windows
   netstat -ano | findstr :4080
   
   # Linux/Mac
   lsof -i :4080
   ```

2. 手动终止进程：
   ```bash
   # Windows
   taskkill /F /PID <pid>
   
   # Linux/Mac
   kill -9 <pid>
   ```

3. 重启计算机

### 权限问题
如果遇到权限问题，请以管理员身份运行终端（Windows）或使用sudo（Linux/Mac）。
