# 📦 条形码商品库存管理系统

女朋友东西太多找不到？别担心，这个系统来救你！

> "我的口红在哪？" "我的那个包包呢？" "我上周买的那个东西去哪了？"
>
> 如果你经常听到这些问题，那么你来对地方了！

一个功能完整的商品库存管理系统，支持通过条形码/二维码扫描记录和管理商品信息。再也不用听"我的口红在哪？"、"我的那个包包呢？"了！

## ✨ 主要功能

- 📷 **条形码/二维码扫描** - 支持扫描商品上的条形码和二维码，再也不用翻箱倒柜找东西
- 📝 **商品信息记录** - 记录商品名称、单价、备注等详细信息，女朋友的每一件宝贝都有档案
- 🖼️ **商品图片上传** - 支持上传商品图片，点击可预览，眼见为实
- 🌐 **网页端管理** - 通过网页查看、添加、修改商品信息，随时随地管理"战利品"
- 🔐 **用户认证** - 支持用户登录和权限管理，保护隐私不被偷看
- 📊 **数据统计** - 查看库存统计和商品列表，知道到底买了多少东西
- 🚀 **一键启动** - 支持npm命令一键启动前后端服务，简单到连男朋友都会用

## 🛠️ 技术栈

### 后端
- Node.js + Express
- TypeScript

- JWT (身份认证)

### 前端
- React 18

## 📦 安装和运行

### 前置要求
- Node.js >= 16
- npm 或 yarn
- Docker 20.10+ (可选，用于Docker部署)

### 安装依赖

```bash
# 方式一：使用npm脚本（推荐）
npm run install:all

# 方式二：手动安装
cd backend
npm install

cd ../frontend
npm install
```

### 运行项目

#### 方式一：Docker 部署（推荐用于生产环境）

```bash
# 构建并启动
docker-compose up -d

# 查看日志
docker-compose logs -f

# 停止服务
docker-compose down
```

访问 http://localhost:4080 开始使用

详细部署说明请查看 [DOCKER_DEPLOYMENT.md](DOCKER_DEPLOYMENT.md)

#### 方式二：本地开发

```bash
# 在项目根目录执行
npm install
npm run init-db
npm run dev
```

#### 方式三：生产构建

```bash
npm run build
npm run start
```

访问 http://localhost:4080 开始使用

### 端口说明

- **统一端口**: 4080
- 前端和后端统一通过 4080 端口访问
- 后端自动托管前端静态文件

## 📱 使用说明

### 1. 扫描商品
- 点击"扫描商品"按钮
- 允许摄像头权限
- 将条形码或二维码对准摄像头
- 系统自动识别并填充商品信息
- 告别"我买了这个但我忘了在哪"的尴尬

### 2. 添加商品
- 填写商品名称、单价、备注等信息
- 可选择上传商品图片
- 点击"保存"按钮
- 商品信息将保存到数据库
- 每一件宝贝都有了自己的"身份证"

### 3. 管理商品
- 在商品列表中查看所有商品
- 点击商品图片可预览大图
- 点击"编辑"修改商品信息
- 点击"删除"移除商品
- 终于可以理直气壮地说"我知道在哪"了
